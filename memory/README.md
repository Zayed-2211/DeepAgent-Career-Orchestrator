# Memory Folder

This folder serves as the **project's long-term memory**. It tracks progress, pending tasks, and architectural decisions to ensure context is never lost.

## Contents

| File | Purpose |
|------|---------|
| [detailed_audit.md](detailed_audit.md) | **Critical** — Full project health report (Bugs, Dead Code, Inconsistencies) |
| [progress_tracking.md](progress_tracking.md) | High-level status of project phases |
| [pending_tasks.md](pending_tasks.md) | Actionable todo list for technical debt and features |
| [architecture_overview.md](architecture_overview.md) | High-level system design and data flow |

## Maintenance

1. **Update `progress_tracking.md`** after every major implementation task.
2. **Move items** from `detailed_audit.md` to `pending_tasks.md` once they are ready to be worked on.
3. **Reference these files** when starting a new session to regain context instantly.
